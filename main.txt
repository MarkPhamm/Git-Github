Git message
 
1. cd (change directory)
2. mkdir (make directory)
3. git init (initialize a Git repository - create a .git)
4. git add (add the specified file(s) into the Git repository, the staging area)
5. git add * (add all files into the staging area)
6. git status (show the modified status of an existing file and the file addition status of a new file, if any, that has to be committed.)
7. git commit -m “message” (commit records or snapshots files permanently in the version history)
8. git remote add origin “[URL]” (pushing our code to the remote (central) repository)
9. git push origin [branch name] (push into Github)
10. git clone [URL] (cloning a git repository)
11. git branch [name-of-the-branch] (creating a new branch)
12. git branch -D [name-of-the-branch] (deleting a branch)
13. git checkout [name-of-the-new-branch] (witch to an existing branch within our repository)
14. git checkout -b [name-of-the-new-branch] (reates a new branch with the given name and switch to that branch)

Delete
15. git rm [file-name] (deleting files locally)
16. git add <deleted-file-name> (staging specified deleted file)
17. git add -u (staging all files)
18. git restore <deleted-file-name>  (restore before commiting the change)
19. git checkout HEAD^ <deleted-file-name> (restore after commiting the change)
